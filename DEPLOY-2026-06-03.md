# 财务中台升级部署说明（2026-06-03）

本次升级包含 8 项中台运营能力，涉及**数据库结构变更**与**新增环境变量**，请按顺序执行。

---

## 一、部署前必做

### 1. 执行数据库迁移（幂等，可重复执行）

迁移脚本：`scripts/2026-06-03-upgrade.sql`

- 给 `user` 表新增 8 列：`bossName`、`city`、`address`、`contactPhone`、`accountStatus`、`subscriptionStartAt`、`pwdPlainEnc`、`pwdUpdatedAt`
- 新建 3 张表：`tenant_pwd_history`（改密历史）、`tenant_renewal`（续费记录）、`tenant_pwd_view_log`（明文查看审计）

在生产 PostgreSQL（118.25.146.225）上执行：

\`\`\`bash
psql "$DATABASE_URL" -v ON_ERROR_STOP=1 -f scripts/2026-06-03-upgrade.sql
\`\`\`

> 脚本全部使用 `ADD COLUMN IF NOT EXISTS` / `CREATE TABLE IF NOT EXISTS`，重复执行安全。

### 2. 配置环境变量 `PWD_ENC_KEY`（密码明文查看功能必需）

用于 AES-256-GCM 加密存储租户登录密码明文副本，供超管在详情页二次验证后查看。

\`\`\`bash
# 生成 32 字节随机密钥
openssl rand -base64 32
\`\`\`

把结果写入服务器环境变量 `PWD_ENC_KEY`（与 `BETTER_AUTH_SECRET` 同级配置）。

**注意事项：**
- **未配置时不会崩溃**：开户流程正常，仅不写入明文副本；查看明文时提示「服务器未配置 PWD_ENC_KEY」。
- **密钥一经设定不可更换**：更换后历史密文将无法解密。请妥善备份。
- 仅对**配置密钥之后**新建或重置密码的租户生成明文副本；存量租户需「重置密码」一次以启用查看。

## 二、保留的既有部署约定（勿遗漏）

以下为历史部署注意事项，本次升级**不改动**，重新部署时仍需确认：

1. **nginx 反向代理**：`/fin` 路径反代规则保持不变。
2. **`tenant_health` 唯一索引**：保持 `tenantId` 唯一索引，健康扫描依赖它做 upsert。
3. **health-scan 定时任务**：保留每日健康扫描 cron。
4. **jiuge 超管账号**：平台超管账号（role=platform）保持可用。

## 三、功能验收清单

部署后逐项确认：

- [ ] 中台「租户管理」列表可点击进入详情页（此前点击 404，本次修复）
- [ ] 详情页四大板块：基本信息（可编辑档案）/ 账号安全 / 订阅管理 / 运营数据
- [ ] 续费：延长到期日并写入 `tenant_renewal`，已到期账号续费后恢复 active
- [ ] 启用/停用：停用后该租户全员登录被拦截，显示「账号已被停用」页
- [ ] 到期拦截：超过到期日 +7 天宽限期后，登录显示「订阅已到期」页
- [ ] 续费横幅：到期前 15 天内 / 宽限期内，客户端顶部显示续费提醒横幅
- [ ] 查看明文密码：超管输入本人密码二次验证后可见，并写入 `tenant_pwd_view_log`
- [ ] 重置密码：可重置租户登录密码，写入 `tenant_pwd_history`
- [ ] 地图：支持「使用中 / 即将到期 / 故障 / 营收 / 客户数」5 维度切换 + 省份脉冲发光点
